# 分析ロジック仕様

## 1. 指標定義
- 勝率 = 勝ちトレード数 / 総トレード数
- 総利益 = 正の損益合計
- 総損失 = 負の損益合計
- 純利益 = 総利益 + 総損失
- PF = 総利益 / abs(総損失)
- 最大ドローダウン = 累積損益曲線の高値から安値までの最大下落

## 2. 集計単位
- リアルタイム集計: 直近表示用（メモリ/Redis）
- 日次確定集計: daily_metrics へ保存
- 任意期間集計: trades テーブルから都度算出

## 3. 時間帯別分析
- exit_time をユーザーのローカル時刻へ変換
- 0〜23時にバケット分け
- それぞれで trades, winRate, netProfit を算出

## 4. 曜日別分析
- exit_time の曜日を算出
- 曜日ごとに trades, winRate, netProfit を算出

## 5. アシスタント寄与分析
- trades.assistant_signal_id が存在するかで採用/非採用を判定
- 採用群と非採用群で以下を比較:
  - trades
  - winRate
  - netProfit
  - averagePnl

## 6. データ品質ルール
- 約定が存在しない注文は分析対象外
- 同一ポジションの部分決済は1トレードに正規化してから集計
- 欠損時刻データは無効扱いにしてエラーログへ送る

## 7. ダッシュボード表示仕様
- 期間セレクタ: 当日 / 直近7日 / 直近30日 / 任意
- KPIカード:
  - 勝率
  - 純利益
  - PF
  - 最大DD
- グラフ:
  - 累積損益折れ線
  - 時間帯ヒートマップ
  - 曜日別棒グラフ
  - AI採用比較バー
