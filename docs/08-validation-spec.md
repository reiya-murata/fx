# 検証仕様書（精度向上・利益最大化）

## 1. 目的
- 戦略が過剰最適化でないことを確認する。
- 実運用に近い条件で期待値の再現性を検証する。

## 2. 検証データ分割
- In-sample: パラメータ探索用
- Out-of-sample: 最終評価用
- Forward test: 疑似リアルタイム評価

## 3. コストモデル
- Bid/Ask スプレッドを反映
- スリッページを確率モデルで反映
- 手数料相当コストを反映

## 4. 検証軸
- 全体成績
- レジーム別成績（TREND_UP / TREND_DOWN / RANGE / HIGH_VOLATILITY）
- 時間帯別成績
- 曜日別成績
- AI採用時/非採用時比較

## 5. 主要KPI
- Expectancy
- Win Rate
- PF
- Max Drawdown
- Recovery Factor
- 平均保有時間
- 連敗最大数

## 6. 合格ライン（初期案）
- Expectancy > 0
- PF >= 1.20
- Max Drawdown <= 10%
- Out-of-sampleでIn-sample比の性能劣化が許容範囲内
- AI採用群が非採用群より net profit で優位

## 7. ウォークフォワード
- 一定期間ごとに再最適化
- 次期間で固定検証
- 複数区間の平均で採否判断

## 8. パラメータ感度
- 主要閾値を±10〜20%変動させて頑健性を確認
- 小さな変化で成績崩壊する戦略は不採用

## 9. 失格条件
- 特定期間だけ極端に収益が偏る
- 高コスト条件でExpectancyが負転
- OOSでPFが1.0未満

## 10. 運用移行ゲート
- 連続する複数検証サイクルで合格
- 失格条件なし
- リスク仕様書の制約をすべて満たす
